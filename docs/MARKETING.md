# Marketing checklist (to help this repo grow)

These are practical steps that tend to move the needle for maker projects.

## 1) Make the landing page irresistible
- Keep the first screen of README to:
  - a one-line value statement
  - 1–2 great photos (already included)
  - a short “features” list
  - “Quick Start” steps

## 2) Add GitHub topics (helps discovery)
Suggested topics:
- esphome
- esp32
- esp32-s3
- hub75
- rgb-matrix
- audio
- fft
- spectrum-analyzer
- home-assistant
- i2s

## 3) Post where your audience already is
- Home Assistant Community Forum (Showcase / Projects)
- Reddit:
  - r/homeassistant
  - r/esp32
  - r/diyelectronics
  - r/led
- Discord servers for ESPHome / HA

## 4) Create a 10–20s clip
What works:
- show boot animation → music kicks in → bars dance
- keep it vertical (phone), post as a short/reel

## 5) Add a Release (even v0.1)
GitHub Releases can increase trust and provide a “download” target.

## 6) Encourage UGC (user generated content)
Add a line in the README:
- “Built one? PR your photo to the Gallery!”

## 7) Add a social preview image
Create an image (1280×640) that includes:
- the panel photo
- project name
- “ESPHome • ESP32‑S3 • HUB75 • INMP441”
Upload it in repo Settings → Social preview.
