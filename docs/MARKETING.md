# Marketing checklist

Practical steps to grow visibility for this project.

## 1) Landing page

Keep the first screen of README to:
- One-line value statement
- 1–2 great photos (hardware photos needed — current ones are from the original Spectrum build)
- Short features list
- Quick Start steps

## 2) GitHub topics

Suggested topics:
- esp32
- esp32-p4
- hub75
- rgb-matrix
- audio
- fft
- spectrum-analyzer
- esp-idf
- led-matrix
- dsp

## 3) Post where your audience is

- Reddit: r/esp32, r/diyelectronics, r/led, r/embedded
- Discord servers for ESP32 / maker communities
- Hackaday.io project page
- Espressif community forums

## 4) Create a 10–20 s clip

What works:
- Show boot animation → music kicks in → 128 bars dance across two panels
- Keep it vertical (phone), post as a short / reel
- Show the physical size of the double-panel display — it's impressive

## 5) Release

Use GitHub Releases with semantic versioning (already set up).
Tag format: `v0.1.0-alpha`, `v1.0.0`, etc.

## 6) Encourage user content

README already includes: "Built a Spectrum Plus? PR your photos to the gallery!"

## 7) Social preview image

Create an image (1280×640) that includes:
- The double-panel display photo (once built)
- Project name
- "Pure C++ Firmware • ESP32-P4 • 128-Band • Dual HUB75 Panels"

Upload in repo Settings → Social preview.
