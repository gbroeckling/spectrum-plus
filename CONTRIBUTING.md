# Contributing

Thanks for wanting to improve this project.

## Quick rules

- **Do not commit secrets.** Never add real `secrets.yaml` values.
- Keep PRs focused: one change set per PR (wiring docs, DSP tuning, rendering, etc.)
- Include photos/logs when reporting hardware issues.

## Good PR ideas

- Add verified wiring for other HUB75 panels (with photos)
- Add enclosure designs (laser-cut / 3D print)
- Add alternate color palettes
- Add a “calibration mode” (optional)

## How to submit

1. Fork the repo
2. Create a branch: `feature/your-change`
3. Commit with clear messages
4. Open a PR describing:
   - hardware used
   - what changed
   - how to reproduce/test
